# [ ] Email implementation

## Description
Implement production email sending with Resend and wire it into Better Auth's email verification flow.

This should send real emails (not console-only) using the verification URL/token generated by Better Auth.

## Requirements
- Email sending:
  - Add email-sending logic in the backend (Convex) using Resend.
  - Keep email logic isolated (e.g. `packages/backend/convex/email/*`).
- Email verification:
  - Implement `sendVerificationEmail` so users receive a verification email containing the real verification link.
  - The email should include:
    - A clear CTA button/link
    - A plain-text fallback link
- Dev vs prod behavior:
  - In development, it's OK to also log the verification URL/token to the terminal to speed up testing.
  - In production, do not log secrets/tokens.

## Acceptance Criteria
- Signing up triggers a verification email via Resend.
- Clicking the link verifies the account and returns the user to the expected callback URL.
- Email sending failures are handled gracefully (clear errors + no app crash).
